## Import and Verify

```python title="Quick Verification"
import vlrdevapi as vlr

# Check if VLR.gg is reachable
if vlr.status.check_status():
    print("Ready to use vlrdevapi!")
else:
    print("VLR.gg is currently unreachable")
```

## Your First Query

<Tabs items={['Matches', 'Search', 'Events']}>
<Tab value="Matches">

```python title="Get Upcoming Matches"
import vlrdevapi as vlr

# Get next 5 upcoming matches
matches = vlr.matches.upcoming(limit=5)

for m in matches:
    print(f"{m.event} - {m.event_phase}")
    print(f"  {m.team1.name} vs {m.team2.name}")
    print(f"  Time: {m.time}")
```

</Tab>
<Tab value="Search">

```python title="Search for Teams and Players"
import vlrdevapi as vlr

# Search across all categories
results = vlr.search.search('sentinels')

print(f"Found {results.total_results} results:")
print(f"  Players: {len(results.players)}")
print(f"  Teams: {len(results.teams)}")
print(f"  Events: {len(results.events)}")
```

</Tab>
<Tab value="Events">

```python title="List Ongoing Events"
import vlrdevapi as vlr
from vlrdevapi.events import EventTier, EventStatus

# Get ongoing VCT events
events = vlr.events.list_events(
    tier=EventTier.VCT,
    status=EventStatus.ONGOING,
    limit=5
)

for e in events:
    print(f"{e.name} - {e.prize or 'TBD'}")
```

## Rate Limiting

By default, vlrdevapi limits requests to **10 per second**. Adjust if needed:

```python title="Configure Rate Limiting"
import vlrdevapi as vlr

# Check current RPS (0.0 means disabled)
print(vlr.get_rate_limit())

# Set to 5 requests/second (enabled)
vlr.configure_rate_limit(requests_per_second=5.0, enabled=True)

# Disable temporarily (not recommended)
vlr.configure_rate_limit(enabled=False)

# Restore default (10 req/s, enabled)
vlr.reset_rate_limit()
```

<Callout type="info" title="Recommended Settings">
Keep rate limiting enabled. The default of 10 requests/second is a good balance for most use cases.
</Callout>
